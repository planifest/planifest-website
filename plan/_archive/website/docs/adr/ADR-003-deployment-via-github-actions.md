# ADR-003: Deployment via GitHub actions to GitHub Pages

## Status
Accepted

## Context
Deploying a static website natively often leverages platforms like Vercel, Netlify, or AWS S3. Providing the public website for a framework requires establishing a cost-effective, easily reproducible deployment pipeline that developers adopting Planifest can easily replicate or understand. Relying on an external third-party vendor simply for hosting a static site introduces unnecessary external dependencies out-of-band of the repository itself.

## Decision
The static HTML output generated by Vite will be deployed natively using the `github-pages` action, triggered on a push or merged PR to the `main` branch. This process generates an artifact that automatically deploys on the `gh-pages` branch.

## Consequences
**Positive:**
- Centralizes infrastructure: No external services (Vercel/Netlify) needed.
- Leverages the robust existing GitHub CDN without added latency or configuration.
- Completely free as part of the public repository features.

**Negative:**
- GitHub pages deploy times can occasionally experience mild queue delays compared to instant Vercel edge deployment propagation.
- Configuration for custom domains directly through GitHub can be slightly less intuitive than purpose-built PaaS frontends.
