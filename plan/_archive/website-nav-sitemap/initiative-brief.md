# Initiative Brief: Website Navigation & Sitemap

## The Product Goal
**What problem are we solving?**
We need a responsive navigation menu and a static sitemap for the Planifest documentation website. The current layout lacked mobile-friendly navigation (hamburger menu) and a comprehensive sitemap indicating where all documents reside.

**Who is this for?**
Users browsing the documentation site on both desktop and mobile devices.

**Why now?**
Navigation is a core requirement for usability. Without a proper responsive menu, mobile users cannot reach internal pages easily. A sitemap is required to expose the breadth of the `.md` documents generated by the Vite builder.

## Strategic Intent
**What are the primary success criteria?**
- A hamburger menu button exists and opens the `.nav-links` menu on mobile devices.
- A static `sitemap.html` is generated organically during the `predev` build step.
- The footer includes a link to the sitemap.
- No heavy pop-ups or modal overlays are used for navigation; keep it lean and native to the page flow.

## The Scope
**What is explicitly out of scope?**
- A complex JavaScript-heavy interactive or search-based modal.
- Backend routing or server-side dynamic sitemap generation (must remain a static site).

## Technical Context
**What existing systems does this touch?**
- `src/web-app/index.html` (the homepage container)
- `src/web-app/src/style.css` (global styles)
- `src/web-app/src/main.ts` (lightweight interaction logic)
- `src/web-app/scripts/build-docs.js` (static generation logic)
- The documentation generator process in the Planifest Framework.
